My Journey Learning Linux

✍️ Aayush Abhyankar 📅 June 2025 🕐 5 min read ☁️ Cloud Computing Project

It started with a broken Windows installation and no recovery drive. I had heard about Linux before — my seniors kept mentioning it, and online forums were full of people swearing by it. Out of desperation more than curiosity, I downloaded Ubuntu, flashed it to a USB drive, and booted into a world I had never seen before. That was the beginning of everything.

The First Week: Confusion and Commands

The first thing Linux teaches you is humility. I had been using Windows for over a decade and suddenly nothing worked the way I expected. Installing software meant typing commands instead of clicking "Next, Next, Finish." Even something as simple as changing the screen brightness required a forum post and three reboots.

But somewhere in that frustration was a spark of genuine curiosity. Every command I ran taught me something. The terminal stopped feeling like a black box and started feeling like a conversation with the operating system itself.

Commands That Changed Everything

These are the first commands I learned — and the ones I still use every single day: 

# Navigate the filesystem
pwd          # print current directory
ls -la       # list files with details
cd /var/log  # change directory

# File operations
cp file.txt backup.txt   # copy
mv old.txt new.txt       # move / rename
rm -rf temp/             # delete (careful!)

# System info
uname -a     # kernel version
df -h        # disk usage
htop         # interactive process viewer

Understanding the Filesystem

One of the biggest mindset shifts was understanding that in Linux, everything is a file. Devices, processes, network sockets — all represented as files in the filesystem. The directory structure made sense once I stopped comparing it to Windows:

  • /etc — system configuration files
  • /home — user personal directories
  • /var — logs and variable data
  • /usr — user-installed programs
  • /proc — virtual filesystem for running processes

Package Management: apt

On Ubuntu and Debian-based distros, apt is your best friend. Instead of hunting for installers online, you simply ask the package manager: 

sudo apt update          # refresh package list
sudo apt upgrade         # upgrade all packages
sudo apt install docker  # install a new package
sudo apt remove vim      # remove a package
"The Linux philosophy is 'laugh in the face of danger'. Oops. Wrong one. 'Do it yourself'. Yes, that's it." — Linus Torvalds

Why Linux Made Me a Better Developer

Learning Linux forced me to understand things I had always taken for granted — how processes work, what environment variables are, how permissions protect files, and why the shell is so powerful. When I later started learning Docker and cloud computing, everything clicked faster because I already understood the Linux underpinnings both tools are built on.

If you're an engineering student who hasn't tried Linux yet, I'd encourage you to start today. Install it in a virtual machine, break things, fix them, and learn. The terminal is intimidating at first, but once it becomes second nature, you'll wonder how you ever managed without it.
