Introduction to Cloud Computing

✍️ Aayush Abhyankar 📅 June 2025 🕐 7 min read ☁️ Cloud Computing Project

Not long ago, running a website meant buying a physical server, setting it up in a room, paying for electricity, and hoping it didn't crash at 2 AM. Today, you can have a globally distributed application running in minutes, paying only for what you use, with built-in redundancy across multiple data centres. That transformation has a name: cloud computing.

What Is Cloud Computing?

Cloud computing is the delivery of computing services — servers, storage, databases, networking, software, and intelligence — over the internet. Instead of owning and maintaining physical hardware, you rent access to these resources from a cloud provider and pay for only what you consume.

The three major cloud providers that dominate the industry are:

  • Microsoft Azure — Deep enterprise integration, strong hybrid cloud
  • Amazon Web Services (AWS) — Largest market share, widest service catalogue
  • Google Cloud Platform (GCP) — Strong in AI/ML and data analytics

The Three Service Models

Cloud services are typically grouped into three models, often visualised as a stack — each layer abstracts more infrastructure away from the user:

IaaS — Infrastructure as a Service

You get raw computing resources: virtual machines, storage, and networking. You manage the OS, middleware, and applications. Examples include Azure Virtual Machines and AWS EC2. This gives maximum control but requires the most management.

PaaS — Platform as a Service

The provider manages the underlying infrastructure and OS. You focus on deploying and managing your application code. Examples include Azure App Service and Google App Engine. Ideal for developers who want to ship fast without worrying about servers.

SaaS — Software as a Service

Fully managed software delivered over the internet. You just use it. Examples include Gmail, Microsoft 365, and Dropbox. No infrastructure or platform management at all.

Deployment Models

Beyond service models, clouds also differ in who can access them:

  • Public Cloud — Resources shared across many customers (e.g. Azure, AWS). Cost-effective and scalable.
  • Private Cloud — Dedicated infrastructure for a single organisation. More control and security, higher cost.
  • Hybrid Cloud — A combination of both. Sensitive data stays on-premises; scalable workloads move to the public cloud.

Core Benefits of Cloud Computing

✔ On-demand self-service    — provision resources instantly, no human interaction
✔ Broad network access      — accessible from anywhere via internet
✔ Resource pooling          — shared infrastructure serves multiple customers
✔ Rapid elasticity          — scale up or down automatically with demand
✔ Measured service          — pay only for what you use (like electricity)

This Project: A Real Cloud Example

This very blog demonstrates cloud computing in action. It is hosted on Azure Static Web Apps — a PaaS offering from Microsoft Azure. The source code lives on GitHub, and every time a change is pushed to the main branch, a GitHub Actions CI/CD pipeline automatically builds and deploys the site to Azure's global CDN.

The result is a website that:

  • Costs nothing to host (Azure free tier)
  • Loads from an edge server closest to the visitor
  • Deploys automatically on every code push
  • Scales infinitely without any configuration
Cloud computing is not about replacing your IT infrastructure — it is about rethinking how software is built, delivered, and maintained in a world where the internet is the platform.

Where to Go From Here

If you're a student exploring cloud computing for the first time, the best next step is hands-on practice. Sign up for the Azure free tier, deploy a static website, spin up a virtual machine, or explore Azure Functions for serverless computing. The concepts only click when you're actually building.

Cloud certifications like the AZ-900 Azure Fundamentals exam are also a great way to validate your knowledge and add credibility to your resume — and Microsoft offers free learning paths to prepare for it.
